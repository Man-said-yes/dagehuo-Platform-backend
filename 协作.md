# 搭伙平台后端项目协作指南

## 项目概述

这是一个搭伙平台的后端项目，主要功能包括：
- 活动发布和管理
- 活动参与和退出
- 群聊功能（包括WebSocket实时通信）
- 用户认证和管理
- 用户信用评价系统
- 活动举报功能
- 系统通知功能（包括WebSocket推送）
- Redis缓存（用于未读消息数和在线用户列表）

## 技术栈

- **后端框架**: Spring Boot 3.5.7
- **数据库**: MySQL
- **缓存**: Redis
- **认证**: JWT
- **WebSockets**: 原生WebSocket实现
- **API文档**: Swagger
- **构建工具**: Maven
- **开发工具**: Trae IDE

## 数据库结构

### 主要表结构

1. **users** - 用户表
   - `id` (Long): 用户ID
   - `openid` (String): 微信openid
   - `student_id` (String): 学号
   - `nickname` (String): 昵称
   - `avatar` (String): 头像URL
   - `gender` (Integer): 性别
   - `phone` (String): 手机号
   - `credit_score` (Integer): 信誉分
   - `high_credit` (Integer): 高信誉分标识
   - `role` (String): 角色（user普通用户，admin管理员）
   - `create_time` (Date): 创建时间
   - `update_time` (Date): 更新时间

2. **activity** - 活动表
   - `id` (Long): 活动ID
   - `title` (String): 活动标题
   - `description` (String): 活动描述
   - `location` (String): 活动地点
   - `event_time` (Date): 活动时间
   - `max_people` (Integer): 最大参与人数
   - `current_people` (Integer): 当前参与人数
   - `status` (Integer): 活动状态（1-招募中，2-进行中，3-已结束，4-已取消）
   - `type` (Integer): 活动类型（0-其他，1-运动，2-约饭，3-学习，4-游戏，5-出行）
   - `creator_id` (Long): 创建者ID
   - `create_time` (Date): 创建时间
   - `update_time` (Date): 更新时间

3. **activity_participant** - 活动参与者表
   - `id` (Long): 主键
   - `activity_id` (Long): 活动ID
   - `user_id` (Long): 用户ID
   - `status` (Integer): 状态（1-已报名）
   - `create_time` (Date): 创建时间

4. **chat_group** - 聊天群表
   - `id` (Long): 群ID
   - `name` (String): 群名称
   - `activity_id` (Long): 关联的活动ID
   - `create_time` (Date): 创建时间

5. **chat_group_member** - 群成员表
   - `id` (Long): 主键
   - `group_id` (Long): 群ID
   - `user_id` (Long): 用户ID
   - `join_time` (Date): 加入时间
   - `last_read_message_id` (Long): 最后已读消息ID

6. **chat_message** - 聊天消息表
   - `id` (Long): 消息ID
   - `group_id` (Long): 群ID
   - `user_id` (Long): 发送者ID
   - `content` (String): 消息内容
   - `type` (Integer): 消息类型（1-文本，2-系统消息）
   - `send_time` (Date): 发送时间

7. **user_credit_record** - 用户信用记录
   - `id` (Long): 记录ID
   - `user_id` (Long): 用户ID
   - `credit_change` (Integer): 信用分变化值（正数加分，负数扣分）
   - `reason` (String): 变化原因（如"靠谱+5"、"鸽子-5"）
   - `create_time` (Date): 创建时间

8. **activity_report** - 活动举报表
   - `id` (Long): 举报记录ID
   - `activity_id` (Long): 活动ID
   - `reporter_user_id` (Long): 举报用户ID
   - `report_reason` (String): 举报理由
   - `report_time` (Date): 举报时间
   - `handle_status` (Integer): 处理状态（0-未处理，1-已核实，2-已驳回）
   - `handle_time` (Date): 处理时间

9. **system_notification** - 系统通知表
   - `id` (Long): 通知ID
   - `user_id` (Long): 接收用户ID
   - `activity_id` (Long): 相关活动ID
   - `notification_type` (Integer): 通知类型（1-活动创建，2-活动结束，3-活动取消，4-新成员加入，5-活动即将开始）
   - `title` (String): 通知标题
   - `content` (String): 通知内容
   - `is_read` (Integer): 是否已读（0-未读，1-已读）
   - `create_time` (Date): 创建时间

## 接口情况

### 已实现的接口

#### 认证管理接口 (`/api/auth`)
- `GET /api/auth/test` - 服务连通性测试
- `POST /api/auth/login` - 微信登录
- `POST /api/auth/bind` - 绑定学号

#### 用户管理接口 (`/api/user`)
- `GET /api/user/info` - 获取当前用户信息
- `GET /api/user/info/{targetUserId}` - 获取指定用户信息
- `PUT /api/user/info` - 更新当前用户信息
- `PUT /api/user/nickname` - 修改当前用户昵称
- `PUT /api/user/phone` - 修改当前用户手机号
- `PUT /api/user/gender` - 修改当前用户性别
- `POST /api/user/avatar` - 上传用户头像
- `PUT /api/user/credit` - 更新当前用户信誉分
- `PUT /api/user/credit/{targetUserId}` - 更新指定用户信誉分
- `POST /api/user/credit/add/{userId}` - 给用户添加信誉分（靠谱）
- `POST /api/user/credit/deduct/{userId}` - 给用户扣除信誉分（鸽子）
- `GET /api/user/credit/history/{userId}` - 查询用户信誉分历史记录
- `GET /api/user/list` - 查询所有用户列表（仅管理员可访问）

#### 活动管理接口 (`/api/event`)
- `POST /api/event` - 发布活动
- `GET /api/event/{eventId}` - 获取活动详情
- `GET /api/event/list` - 获取活动列表（支持按时间正序和倒序）
- `GET /api/event/my` - 获取我的活动
- `GET /api/event/participated` - 获取我参与的活动
- `PUT /api/event/{eventId}` - 更新活动
- `DELETE /api/event/{eventId}` - 删除活动
- `POST /api/event/{eventId}/join` - 加入活动
- `POST /api/event/{eventId}/exit` - 退出活动
- `PUT /api/event/{eventId}/status` - 更新活动状态
- `POST /api/event/list/nearby` - 根据距离查询活动列表（由近及远）
- `POST /api/event/list/distant` - 根据距离查询活动列表（由远及近）
- `GET /api/event/{eventId}/participants` - 获取活动参与者详情
- `POST /api/event/{eventId}/report` - 举报活动
- `GET /api/event/report/list` - 获取被举报活动列表
- `GET /api/event/report/list/status` - 根据处理状态获取被举报活动列表
- `POST /api/event/{eventId}/cancel` - 取消活动
- `POST /api/event/{eventId}/end` - 结束活动

#### 聊天管理接口 (`/api/chat`)
- `GET /api/chat/groups` - 获取用户群聊列表
- `GET /api/chat/group/{groupId}` - 获取群聊详情
- `GET /api/chat/group/{groupId}/members` - 获取群聊成员列表
- `GET /api/chat/group/{groupId}/messages` - 获取群聊消息
- `POST /api/chat/group/{groupId}/message` - 发送消息
- `POST /api/chat/group/{groupId}/read` - 标记消息已读
- `POST /api/chat/group/{groupId}/member` - 添加群成员
- `DELETE /api/chat/group/{groupId}/member/{userId}` - 移除群成员
- `GET /api/chat/group/{groupId}/online` - 获取群在线用户列表

#### 系统通知接口 (`/api/notification`)
- `GET /api/notification/list` - 获取用户的系统通知列表
- `GET /api/notification/list/type` - 根据类型获取系统通知
- `GET /api/notification/unread/count` - 获取未读通知数量
- `PUT /api/notification/{notificationId}/read` - 标记通知为已读
- `PUT /api/notification/read/batch` - 批量标记通知为已读

#### WebSocket接口
- `ws://localhost:8080/ws/chat?token=YOUR_TOKEN` - 实时聊天连接（支持系统通知推送）

## 开发环境搭建

### 1. 环境要求
- JDK 17+
- Maven 3.8+
- MySQL 8.0+
- Redis 6.0+
- Trae IDE

### 2. 项目导入
1. 在Trae IDE中打开项目目录
2. 等待Maven依赖下载完成
3. 配置数据库连接（确保MySQL和Redis服务已启动）

### 3. 数据库初始化
项目中已包含SQL初始化脚本，位于 `src/main/resources/sql/init.sql`

### 4. 运行项目
- 开发环境：`mvn spring-boot:run`
- 测试环境：`mvn spring-boot:run -Dspring.profiles.active=test`

## 工作流程

### 1. 分支管理
- 使用主分支 `main` 进行开发
- 重要功能可以创建特性分支

### 2. 代码规范
- 遵循Java编码规范
- 类名使用大驼峰命名法
- 方法名和变量名使用小驼峰命名法
- 常量使用全大写加下划线
- 适当添加注释

### 3. 提交规范
- 提交信息要清晰明了
- 一次提交只包含一个功能或修复
- 提交前确保代码能正常编译和运行

### 4. 测试要求
- 新增功能要编写测试用例
- 确保现有功能不受影响
- 运行完整的测试套件

## 注意事项

1. **数据库连接**：确保MySQL和Redis服务正常运行
2. **端口冲突**：默认使用8080端口，若被占用需修改配置
3. **JWT令牌**：所有需要认证的接口都需要在请求头中携带JWT令牌
4. **WebSocket连接**：连接时需要携带token参数
5. **异常处理**：统一使用Result类返回错误信息
6. **日志记录**：重要操作要记录日志
7. **性能优化**：注意Redis缓存的使用，避免频繁数据库操作
8. **安全性**：注意SQL注入防护，使用参数化查询
