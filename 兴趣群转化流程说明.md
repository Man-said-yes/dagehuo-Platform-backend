# 兴趣群转化流程说明

## 一、什么是兴趣群转化？

当活动结束后，如果参与人数大于2人，系统会自动将活动参与者转化为一个长期存在的"兴趣群"，方便大家继续交流和组局。

---

## 二、两种创建兴趣群的方式

### 方式一：自动创建（推荐）⭐

**适用场景**：活动结束时自动转化

**触发条件**：活动创建者调用"结束活动"接口

**优点**：
- ✅ 全自动，无需前端额外处理
- ✅ 自动将所有参与者加入兴趣群
- ✅ 自动设置群主为活动创建者

**流程**：
```
活动进行中
    ↓
创建者点击"结束活动"
    ↓
前端调用：POST /api/event/{eventId}/end
    ↓
后端自动处理：
    1. 活动状态 → 已结束（status=3）
    2. 活动群聊 → 已解散（status=2）
    3. 检查参与人数
        ├─ > 2人 → 自动创建兴趣群
        │          ├─ 群类型：兴趣群（type=2）
        │          ├─ 群名称：{活动标题}兴趣群
        │          ├─ 群主：活动创建者
        │          └─ 成员：所有参与者自动加入
        └─ ≤ 2人 → 不创建兴趣群
    ↓
用户在群聊列表看到兴趣群
```

### 方式二：手动创建

**适用场景**：前端主动创建兴趣群

**接口**：`POST /api/chat/interest-group`

**请求参数**：
```json
{
  "activityId": 1,        // 活动ID
  "groupName": "羽毛球兴趣群",  // 群名称
  "ownerId": 1            // 群主ID
}
```

**使用场景**：
- 用户想手动创建兴趣群
- 不依赖活动结束
- 需要自定义群名称和群主

**注意**：
- ⚠️ 手动创建后，需要前端自己调用添加成员接口
- ⚠️ 不会自动添加参与者

---

## 三、推荐使用方式

### 🎯 推荐：使用自动创建

**原因**：
1. **无需额外代码**：前端只需调用结束活动接口
2. **自动添加成员**：所有参与者自动加入兴趣群
3. **数据一致性**：避免重复创建群聊

**前端代码示例**：
```javascript
// 结束活动
async endActivity() {
  try {
    const res = await uni.showModal({
      title: '确认结束活动？',
      content: '结束后将自动创建兴趣群（参与人数>2人）'
    })
    
    if (res.confirm) {
      // 只需调用这一个接口
      await this.$api.activity.end(this.activityId)
      
      uni.showToast({
        title: '活动已结束',
        icon: 'success'
      })
      
      // 后端已自动创建兴趣群，无需额外处理
      this.loadActivityDetail()
    }
  } catch (e) {
    uni.showToast({
      title: e.msg || '操作失败',
      icon: 'none'
    })
  }
}
```

### 📌 手动创建的使用场景

**场景1：测试时手动创建**
```javascript
// 测试时可以手动创建兴趣群
async createInterestGroup() {
  try {
    const data = await this.$api.chat.createInterestGroup({
      activityId: this.activityId,

    })
    
    // 手动添加成员
    await this.$api.chat.addMember({
      groupId: data.id,
      userId: this.memberId
    })
    
    uni.showToast({
      title: '创建成功',
      icon: 'success'
    })
  } catch (e) {
    uni.showToast({
      title: e.msg || '创建失败',
      icon: 'none'
    })
  }
}
```

**场景2：特殊需求**
- 用户想创建一个与活动无关的兴趣群
- 需要自定义群名称
- 需要手动控制成员加入

## 四、前端需要做什么？

### 1. 活动详情页 - 结束活动按钮

```vue
<template>
  <view class="activity-detail">
    <!-- 活动信息 -->
    <uni-card>
      <view class="info-item">
        <text>活动标题：{{activity.title}}</text>
      </view>
      <view class="info-item">
        <text>活动状态：{{statusText}}</text>
      </view>
      <view class="info-item">
        <text>参与人数：{{activity.currentPeople}}/{{activity.maxPeople}}</text>
      </view>
    </uni-card>
    
    <!-- 操作按钮 -->
    <view class="action-buttons">
      <!-- 创建者且活动进行中，显示结束按钮 -->
      <uni-button 
        v-if="isCreator && activity.status === 1" 
        @click="endActivity"
        type="warn"
      >
        结束活动
      </uni-button>
      
      <!-- 非创建者且未加入，显示加入按钮 -->
      <uni-button 
        v-if="!isCreator && !isJoined && activity.status === 1" 
        @click="joinActivity"
        type="primary"
      >
        加入活动
      </uni-button>
      
      <!-- 已加入，显示退出按钮 -->
      <uni-button 
        v-if="isJoined && activity.status === 1" 
        @click="exitActivity"
        type="default"
      >
        退出活动
      </uni-button>
    </view>
  </view>
</template>

<script>
export default {
  data() {
    return {
      activityId: null,
      activity: {},
      currentUserId: null
    }
  },
  computed: {
    isCreator() {
      return this.activity.creatorId === this.currentUserId
    },
    isJoined() {
      // 判断当前用户是否已加入
      return this.activity.participants?.some(p => p.userId === this.currentUserId)
    },
    statusText() {
      const statusMap = {
        1: '招募中',
        2: '进行中',
        3: '已结束',
        4: '已取消'
      }
      return statusMap[this.activity.status]
    }
  },
  methods: {
    // 结束活动
    async endActivity() {
      try {
        // 弹出确认框
        const res = await uni.showModal({
          title: '确认结束活动？',
          content: '结束后将自动创建兴趣群（参与人数>2人），活动群聊将解散'
        })
        
        if (res.confirm) {
          // 调用结束活动接口
          await this.$api.activity.end(this.activityId)
          
          uni.showToast({
            title: '活动已结束',
            icon: 'success'
          })
          
          // 刷新活动详情
          await this.loadActivityDetail()
          
          // 提示兴趣群创建
          if (this.activity.currentPeople > 2) {
            setTimeout(() => {
              uni.showModal({
                title: '兴趣群已创建',
                content: '兴趣群已自动创建，您可以在群聊列表中查看',
                showCancel: false
              })
            }, 1500)
          }
        }
      } catch (e) {
        uni.showToast({
          title: e.msg || '操作失败',
          icon: 'none'
        })
      }
    },
    
    // 加载活动详情
    async loadActivityDetail() {
      try {
        const data = await this.$api.activity.getDetail(this.activityId)
        this.activity = data
      } catch (e) {
        uni.showToast({
          title: '加载失败',
          icon: 'none'
        })
      }
    }
  },
  onLoad(options) {
    this.activityId = options.id
    this.currentUserId = uni.getStorageSync('userId')
    this.loadActivityDetail()
  }
}
</script>
```

### 2. 群聊列表页 - 显示不同类型的群

```vue
<template>
  <view class="chat-list">
    <!-- 群聊列表 -->
    <view 
      class="chat-item" 
      v-for="group in activeGroups" 
      :key="group.id"
      @click="enterChat(group.id)"
    >
      <view class="group-avatar">
        <image :src="group.avatar || '/static/default-group.png'" />
      </view>
      
      <view class="group-info">
        <view class="group-name">
          <text>{{group.groupName}}</text>
          <!-- 群类型标签 -->
          <uni-tag 
            :text="group.type === 1 ? '活动群' : '兴趣群'" 
            :type="group.type === 1 ? 'primary' : 'success'"
            size="small"
          />
        </view>
        
        <view class="last-message">
          <text>{{group.lastMessage || '暂无消息'}}</text>
        </view>
      </view>
      
      <view class="group-meta">
        <view class="time">{{formatTime(group.lastMessageTime)}}</view>
        <uni-badge 
          v-if="group.unreadCount > 0" 
          :text="group.unreadCount" 
          absolute
        />
      </view>
    </view>
    
    <!-- 空状态 -->
    <uni-empty v-if="activeGroups.length === 0" text="暂无群聊" />
  </view>
</template>

<script>
export default {
  data() {
    return {
      groupList: []
    }
  },
  computed: {
    // 过滤已解散的群
    activeGroups() {
      return this.groupList.filter(g => g.status === 1)
    }
  },
  methods: {
    // 加载群聊列表
    async loadGroupList() {
      try {
        const data = await this.$api.chat.getGroups()
        this.groupList = data
      } catch (e) {
        uni.showToast({
          title: '加载失败',
          icon: 'none'
        })
      }
    },
    
    // 进入群聊
    enterChat(groupId) {
      uni.navigateTo({
        url: `/pages/chat/room?groupId=${groupId}`
      })
    },
    
    // 格式化时间
    formatTime(time) {
      if (!time) return ''
      const date = new Date(time)
      const now = new Date()
      const diff = now - date
      
      if (diff < 60000) return '刚刚'
      if (diff < 3600000) return Math.floor(diff / 60000) + '分钟前'
      if (diff < 86400000) return Math.floor(diff / 3600000) + '小时前'
      return date.toLocaleDateString()
    }
  },
  onShow() {
    this.loadGroupList()
  }
}
</script>
```

## 五、接口说明

### 1. 结束活动接口

**请求**：
```javascript
POST /api/event/{eventId}/end
```

**响应**：
```json
{
  "code": 200,
  "msg": "success",
  "data": "活动已结束"
}
```

**后端自动处理**：
1. 更新活动状态为"已结束"
2. 解散活动群聊
3. 创建兴趣群（如果参与人数>2人）
4. 将所有参与者加入兴趣群

### 2. 获取群聊列表接口

**请求**：
```javascript
GET /api/chat/groups
```

**响应**：
```json
{
  "code": 200,
  "msg": "success",
  "data": [
    {
      "groupId": 1,
      "groupName": "羽毛球活动-聊天群",
      "type": 1,              // 1-活动群 2-兴趣群
      "status": 2,            // 1-正常 2-解散
      "activityId": 1,
      "lastMessage": "大家好",
      "lastMessageTime": "2024-01-01T19:00:00",
      "unreadCount": 0
    },
    {
      "groupId": 2,
```

## 六、常见问题

### Q1: 兴趣群什么时候创建？
**A**: 活动创建者点击"结束活动"后，后端自动创建。前端无需调用额外接口。

### Q2: 兴趣群的群主是谁？
**A**: 兴趣群的群主是原活动的创建者。

### Q3: 参与人数≤2人怎么办？
**A**: 不会创建兴趣群，只有活动群聊会被解散。

### Q4: 用户如何知道兴趣群已创建？
**A**: 
1. 结束活动后，前端可以弹窗提示
2. 用户在群聊列表中可以看到新的兴趣群
3. 可以通过系统通知告知用户

### Q5: 兴趣群可以解散吗？
**A**: 可以，群主可以调用解散接口：
```javascript
POST /api/chat/group/{groupId}/dissolve
```

### Q6: 活动群聊和兴趣群有什么区别？
**A**: 
- **活动群聊**：活动创建时自动创建，活动结束时自动解散
- **兴趣群**：活动结束后创建，长期存在，群主可以手动解散

## 七、测试流程

1. 创建一个活动，邀请3人以上参加
2. 以创建者身份登录，点击"结束活动"
3. 查看群聊列表，应该看到：
   - 原活动群聊显示"已解散"
   - 新的兴趣群显示"兴趣群"标签
4. 进入兴趣群，可以正常发送消息
5. 所有参与者都应该在兴趣群中

## 八、注意事项

1. **前端无需额外处理**：兴趣群创建完全由后端自动完成
2. **过滤已解散群聊**：在群聊列表中过滤 `status === 2` 的群
3. **区分群类型**：通过 `type` 字段区分活动群和兴趣群
4. **用户提示**：结束活动后建议提示用户兴趣群已创建
5. **权限控制**：只有活动创建者可以结束活动
