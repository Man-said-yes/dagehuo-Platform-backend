# 搭个伙小程序前端开发建议

## 项目概述

**项目名称**：搭个伙 - 校园活动搭伙平台  
**剩余时间**：10天  
**技术栈**：uniapp + 微信小程序  
**后端地址**：`http://your-server-ip:8080`

## 二、页面功能与接口对应关系

### 1. 活动首页（核心页面）

**页面路径**：`pages/activity/index`

**功能说明**：
- 展示正在招募的活动列表
- 支持按类型筛选（运动、学习、娱乐、其他）
- 支持按距离排序
- 支持搜索活动

**需要调用的接口**：

```javascript
// 1. 获取活动列表（分页）
GET /api/event/list?page=1&pageSize=10&sortBy=create_time&order=desc

// 2. 按类型获取活动
GET /api/event/list?type=1&page=1&pageSize=10  // type: 1运动 2学习 3娱乐 4其他

// 3. 按距离排序（需要用户授权位置）
GET /api/event/list/nearby?longitude=113.123&latitude=23.456&type=1&page=1&pageSize=10

// 4. 搜索活动
GET /api/event/search?keyword=羽毛球

// 5. 获取活动总数
GET /api/event/count?type=1
```

**开发建议**：
- 使用`<scroll-view>`实现下拉刷新和上拉加载
- 使用`<uni-segmented-control>`实现类型切换
- 缓存活动列表数据，减少重复请求
- 使用`<uni-load-more>`显示加载状态

**数据缓存策略**：
```javascript
// 在onLoad时先读取缓存
const cachedActivities = uni.getStorageSync('activities')
if (cachedActivities) {
  this.activityList = cachedActivities
}

// 请求成功后更新缓存
uni.setStorageSync('activities', this.activityList)
```

**页面路径**：`pages/activity/publish`

```javascript

  "type": 1,              // 活动类型
```

### 3. 活动详情页

**页面路径**：`pages/activity/detail`

**功能说明**：
- 查看活动详细信息
- 查看参与者列表
- 加入/退出活动
- 查看群聊入口
- 举报活动
- 结束活动（创建者）

**兴趣群转化说明**：
- 活动创建者可以点击"结束活动"按钮
- 结束活动后，如果参与人数>2人，后端会自动创建兴趣群
- 兴趣群会自动出现在参与者的群聊列表中
- 前端无需额外处理，只需调用结束活动接口即可

**需要调用的接口**：

```javascript
// 1. 获取活动详情
GET /api/event/{eventId}

// 2. 获取参与者列表
GET /api/event/{eventId}/participants

// 3. 加入活动
POST /api/event/{eventId}/join

// 4. 退出活动
POST /api/event/{eventId}/exit

// 5. 举报活动
POST /api/event/{eventId}/report
{
  "reportReason": "违规内容"
}

// 6. 取消活动（仅创建者）
POST /api/event/{eventId}/cancel

// 7. 结束活动（仅创建者）
POST /api/event/{eventId}/end
```

**开发建议**：
- 使用`<uni-card>`展示活动信息
- 使用`<uni-grid>`展示参与者头像
- 加入按钮需要判断：是否已加入、是否已满、是否已结束
- 创建者显示"取消活动"和"结束活动"按钮
- 使用`<uni-popup>`显示参与者列表

### 4. 群聊页

**页面路径**：`pages/chat/index` 和 `pages/chat/room`

**功能说明**：
- 查看我的群聊列表
- 进入群聊发送消息
- 查看群成员
- 查看群详情

**群聊类型说明**：
- **活动群聊**（type=1）：活动创建时自动创建，活动结束时自动解散
- **兴趣群**（type=2）：活动结束后自动转化，参与人数>2人时创建

**兴趣群转化流程**：
1. 活动创建者点击"结束活动"
2. 后端自动将活动群聊标记为"解散"（status=2）
3. 后端自动创建兴趣群（如果参与人数>2人）
4. 所有参与者自动加入兴趣群
5. 兴趣群出现在用户的群聊列表中

**需要调用的接口**：

```javascript
// 1. 获取我的群聊列表
GET /api/chat/groups

// 2. 获取群聊详情
GET /api/chat/group/{groupId}

// 3. 获取群成员列表
GET /api/chat/group/{groupId}/members

// 4. 获取群聊消息（分页）
GET /api/chat/group/{groupId}/messages?page=1&pageSize=20

// 5. 发送消息
POST /api/chat/group/{groupId}/message
{
  "content": "大家好",
  "type": 1  // 1文本 2图片
}

// 6. 标记消息已读
POST /api/chat/group/{groupId}/read
{
  "lastReadMessageId": 10
}

// 7. 获取在线用户
GET /api/chat/group/{groupId}/online

// 8. 解散群聊（仅群主）
POST /api/chat/group/{groupId}/dissolve
```

**WebSocket连接**：
```javascript
// 连接WebSocket
const socketTask = uni.connectSocket({
  url: 'ws://your-server-ip:8080/ws/chat?token=' + token,
  success: () => {
    console.log('WebSocket连接成功')
  }
})

// 接收消息
socketTask.onMessage((res) => {
  const message = JSON.parse(res.data)
  // 处理收到的消息
})

// 发送消息
socketTask.send({
  data: JSON.stringify({
    type: 'chat',
    groupId: 1,
    content: '大家好'
  })
})
```

**开发建议**：
- 使用`<scroll-view>`实现消息列表滚动
- 使用`<uni-badge>`显示未读消息数
- WebSocket断线自动重连
- 消息发送失败重试机制
- 图片消息使用`<uni-file-picker>`上传

## 三、快速开发技巧

### 1. 封装请求工具

```javascript
// utils/request.js
const BASE_URL = 'http://your-server-ip:8080'

export function request(options) {
  const token = uni.getStorageSync('token')
  
  return new Promise((resolve, reject) => {
    uni.request({
      url: BASE_URL + options.url,
      method: options.method || 'GET',
      data: options.data,
      header: {
        'Content-Type': 'application/json',
        'Authorization': token ? `Bearer ${token}` : ''
      },
      success: (res) => {
        if (res.data.code === 200) {
          resolve(res.data.data)
        } else if (res.data.code === 401) {
          // token过期，跳转登录
          uni.navigateTo({ url: '/pages/login/login' })
        } else {
          uni.showToast({
            title: res.data.msg || '请求失败',
            icon: 'none'
          })
          reject(res.data)
        }
      },
      fail: reject
    })
  })
}
```

### 2. 封装API接口

```javascript
// api/activity.js
import { request } from '@/utils/request'

export default {
  // 获取活动列表
  getList(params) {
    return request({
      url: '/api/event/list',
      data: params
    })
  },
  
  // 发布活动
  create(data) {
    return request({
      url: '/api/event',
      method: 'POST',
      data
    })
  },
  
  // 获取活动详情
  getDetail(id) {
    return request({
      url: `/api/event/${id}`
    })
  },
  
  // 加入活动
  join(id) {
    return request({
      url: `/api/event/${id}/join`,
      method: 'POST'
    })
  }
}
```

### 3. 使用全局状态管理

```javascript
// store/index.js
import Vue from 'vue'
import Vuex from 'vuex'

Vue.use(Vuex)

export default new Vuex.Store({
  state: {
    userInfo: null,
    token: uni.getStorageSync('token') || ''
  },
  mutations: {
    SET_USER_INFO(state, userInfo) {
      state.userInfo = userInfo
    },
    SET_TOKEN(state, token) {
      state.token = token
      uni.setStorageSync('token', token)
    }
  }
})
```

### 4. 使用mixins复用代码

```javascript
// mixins/list.js
export default {
  data() {
    return {
      list: [],
      page: 1,
      pageSize: 10,
      hasMore: true,
      loading: false
    }
  },
  methods: {
    async loadMore() {
      if (!this.hasMore || this.loading) return
      
      this.loading = true
      this.page++
      
      try {
        const data = await this.fetchList()
        this.list = [...this.list, ...data]
        this.hasMore = data.length === this.pageSize
      } finally {
        this.loading = false
      }
    }
  }
}
```
