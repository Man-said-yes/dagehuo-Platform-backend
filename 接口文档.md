# 搭伙平台后端接口文档

## 环境配置说明

### 1. 环境切换

本项目支持两种运行环境：

- **开发环境（dev）**: 使用MySQL数据库
- **测试环境（test）**: 使用H2内存数据库（无需额外配置，适合前端直接测试）

### 2. 如何切换环境

#### 方法一：使用Spring Boot profiles

在启动应用时添加 `-Dspring.profiles.active` 参数：

```bash
# 使用开发环境（MySQL）
mvn spring-boot:run

# 使用测试环境（H2内存数据库）
mvn spring-boot:run -Dspring.profiles.active=test
```

#### 方法二：修改application.yml

在 `application.yml` 文件中修改 `spring.profiles.active` 属性：

```yaml
spring:
  profiles:
    active: test  # 切换到测试环境
```

### 3. 测试环境特性

- **H2内存数据库**: 无需安装MySQL，启动即可使用
- **自动初始化**: 启动时自动创建表结构并插入测试数据
- **H2控制台**: 可通过 `http://localhost:8080/h2-console` 访问数据库
  - **JDBC URL**: `jdbc:h2:mem:dagehuo`
  - **用户名**: `sa`
  - **密码**: 空

- **测试账号**: 系统已内置3个测试用户，可直接使用以下openid登录：
  - `test_openid_001` (张三)
  - `test_openid_002` (李四)
  - `test_openid_003` (王五)

## 接口概览

### 认证管理接口 (`/api/auth`)
- `GET /api/auth/test` - 服务连通性测试
- `POST /api/auth/login` - 微信登录
- `POST /api/auth/bind` - 绑定学号

### 2. 用户管理接口 (`/api/user`)
- `GET /api/user/info` - 获取当前用户信息
- `GET /api/user/info/{targetUserId}` - 获取指定用户信息 管理员接口
- `PUT /api/user/info` - 更新当前用户信息
- `PUT /api/user/nickname` - 修改当前用户昵称
- `PUT /api/user/credit/{targetUserId}` - 更新用户信誉分 管理员接口

### 活动管理接口 (`/api/event`)
- `POST /api/event` - 发布活动
- `GET /api/event/{eventId}` - 获取活动详情 
- `GET /api/event/list` - 获取活动列表
- `GET /api/event/my` - 获取我的活动
- `GET /api/event/participated` - 获取我参与的活动
- `PUT /api/event/{eventId}` - 更新活动
- `DELETE /api/event/{eventId}` - 删除活动
- `POST /api/event/{eventId}/join` - 加入活动
- `POST /api/event/{eventId}/exit` - 退出活动

### 聊天管理接口 (`/api/chat`)
- `GET /api/chat/groups` - 获取用户群聊列表
- `GET /api/chat/group/{groupId}` - 获取群聊详情
- `GET /api/chat/group/{groupId}/members` - 获取群聊成员列表
- `GET /api/chat/group/{groupId}/messages` - 获取群聊消息
- `POST /api/chat/group/{groupId}/message` - 发送消息
- `POST /api/chat/group/{groupId}/read` - 标记消息已读
- `POST /api/chat/group/{groupId}/member` - 添加群成员
- `DELETE /api/chat/group/{groupId}/member/{userId}` - 移除群成员

### WebSocket 连接
- `ws://localhost:8080/ws/chat?token=YOUR_TOKEN` - 实时聊天连接

## 配置说明

### 1. 微信小程序配置

在 `application.yaml` 中配置：

```yaml
wechat:
  app-id: your_app_id_here    # 替换为你的微信小程序appId
  app-secret: your_app_secret_here  # 替换为你的微信小程序appSecret
  test-mode: true  # 测试模式：true-使用code作为openid（开发测试用），false-调用真实微信接口

jwt:
  secret: dagehuo-platform-secret-key-2024  # JWT密钥
  expiration: 86400000  # Token过期时间（毫秒，默认24小时）
```

### 2. 测试模式说明

开发测试时，可以开启测试模式，无需真实的微信小程序配置：

- **test-mode: true** - 测试模式：code 直接作为 openid 使用
- **test-mode: false** - 生产模式：调用真实微信接口获取 openid

**测试模式使用方法**：
调用登录接口时，code 可以填任意字符串，例如：
- `test_openid_001`
- `user_123`
- `demo_user`

## 前端调用示例

### 1. 微信小程序登录流程

```javascript
// 1. 调用微信登录接口获取code
wx.login({
  success: (res) => {
    if (res.code) {
      // 2. 将code发送到后端
      wx.request({
        url: 'http://localhost:8080/api/auth/login',
        method: 'POST',
        data: {
          code: res.code
        },
        success: (response) => {
          if (response.data.code === 200) {
            const { token, registered, studentId, userId } = response.data.data;
            
            // 3. 保存token
            wx.setStorageSync('token', token);
            
            // 4. 判断是否需要绑定学号
            if (registered) {
              // 已绑定，跳转到首页
              wx.switchTab({ url: '/pages/index/index' });
            } else {
              // 未绑定，跳转到绑定页面
              wx.navigateTo({ url: '/pages/bind/bind' });
            }
          }
        }
      });
    }
  }
});
```

### 2. 绑定学号

```javascript
// 从token中获取openid，或者直接使用登录时保存的
const openid = '...';

wx.request({
  url: 'http://localhost:8080/api/auth/bind',
  method: 'POST',
  data: {
    openid: openid,
    studentId: '24000000000'
  },
  success: (response) => {
    if (response.data.code === 200) {
      wx.showToast({
        title: '绑定成功',
        icon: 'success'
      });
      // 跳转到完善信息页面
      wx.navigateTo({ url: '/pages/profile/profile' });
    } else {
      wx.showToast({
        title: response.data.msg,
        icon: 'none'
      });
    }
  }
});
```

## WebSocket 实时聊天

### 连接方式

**连接地址**:
```
ws://localhost:8080/ws/chat?token=YOUR_TOKEN
```

**说明**:
- 需要携带JWT token作为查询参数
- 连接成功后，自动加入用户所在的所有群聊的在线用户列表
- 断开连接后，自动从所有群聊的在线用户列表中移除

### 微信小程序接入示例

```javascript
// 1. 连接WebSocket
wx.connectSocket({
  url: 'ws://localhost:8080/ws/chat?token=' + wx.getStorageSync('token'),
  success: function() {
    console.log('WebSocket连接成功');
  }
});

// 2. 监听连接打开
wx.onSocketOpen(function() {
  console.log('WebSocket连接已打开');
});

// 3. 监听服务器消息
wx.onSocketMessage(function(res) {
  const message = JSON.parse(res.data);
  console.log('收到服务器消息:', message);
  
  if (message.type === 'chat_message') {
    // 处理聊天消息
    handleChatMessage(message);
  } else if (message.type === 'user_status') {
    // 处理用户上下线
    handleUserStatus(message);
  }
});

// 4. 发送消息
function sendMessage(groupId, content) {
  const message = {
    groupId: groupId,
    content: content
  };
  wx.sendSocketMessage({
    data: JSON.stringify(message),
    success: function() {
      console.log('消息发送成功');
    }
  });
}

// 5. 监听连接关闭
wx.onSocketClose(function() {
  console.log('WebSocket连接已关闭');
  // 可以在这里进行重连
});

// 6. 监听错误
wx.onSocketError(function(res) {
  console.error('WebSocket连接错误:', res);
});
```

### Redis 存储说明

**1. 在线用户列表**
- 数据结构：Set
- Key格式：`online:{groupId}`
- 存储内容：用户ID列表

**2. 未读消息数**
- 数据结构：Hash
- Key格式：`unread:{userId}`
- 存储内容：groupId -> unreadCount

**3. WebSocket会话映射**
- 数据结构：Hash
- Key格式：`ws:session`
- 存储内容：userId -> sessionId

**4. 聊天消息**
- 数据结构：Hash
- Key格式：`message:{groupId}:{timestamp}`
- 存储内容：userId, content, sendTime

### 发送消息说明

**推荐使用WebSocket方式发送消息**，这是完整的解决方案，既保证实时性，又保证消息持久化：

#### WebSocket发送消息（推荐）

**优点**：
- 实时性好，消息立即广播给所有在线用户
- 消息同时存储到数据库和Redis
- 自动增加未读消息数
- 减少HTTP请求开销

**使用场景**：
- 实时聊天，发送消息后立即显示
- 需要消息历史记录

### 工作流程

**WebSocket发送消息流程**:
1. 小程序通过WebSocket发送消息
2. 后端接收消息并解析
3. 后端将消息存储到数据库
4. 后端为群内其他成员增加未读消息数
5. 后端将消息广播给群内所有在线用户
6. 小程序接收广播消息并显示

**用户上线流程**:
1. 小程序连接WebSocket
2. 后端验证token并获取用户信息
3. 后端将用户添加到所有所在群的在线列表
4. 后端广播用户上线消息给群内所有在线用户

**用户下线流程**:
1. 小程序断开WebSocket连接
2. 后端从所有所在群的在线列表中移除用户
3. 后端广播用户下线消息给群内所有在线用户

### 推荐使用方式

**对于实时聊天场景**：
1. 使用WebSocket发送消息
2. 进入群聊时调用REST接口获取历史消息

## 注意事项

1. **跨域支持**: 所有接口已配置 `@CrossOrigin(origins = "*")`，支持跨域请求
2. **学号格式**: 必须以 "24" 开头，且为11位数字
3. **学号唯一性**: 同一学号只能被一个微信账号绑定
4. **参数验证**: 所有必填参数都会进行格式验证
5. **Token有效期**: 默认24小时，过期需要重新登录
6. **微信配置**: 需要在 application.yaml 中正确配置 appId 和 appSecret
7. **活动权限**: 只有活动创建者可以更新或删除活动
8. **WebSocket连接**: 需要在连接时携带JWT token作为查询参数
